<IT基本用語>

■ IT
IT ・・・インターネットなどの通信とコンピューターを駆使する情報技術。
ICT ・・・通信技術を使って人とインターネット、人と人がつながる技術のこと。
IoT ・・・人を使わずモノが自動的にインターネットとつながる技術のこと。

■ OS
OS ・・・コンピュータの入出力や同時並行処理などを管理するプログラム。
Windows ・・・マイクロソフトが開発・販売しているOS。
macOS (Mac OS X) ・・・Appleが開発・販売しているOS。
UNIX ・・・現存するもっとも古いOSのひとつ。コマンド画面で操作する。
Linux ・・・UNIXの親戚のようなOS。オープンソースなので自由に使える。

■ 言語・環境
マークアップ／フロントエンド ・・・WEBサービスやWEBアプリケーションで直接ユーザーの目に触れる部分のこと。
HTML ・・・静的なWEBページを制作する言語。ほとんどのWEBページで使用されている。
CSS ・・・HTMLと組み合わせて使用する言語。WEBページの装飾に使用する。
JavaScript ・・・動的なWEBページを生成することができるプログラミング言語。クライアントサイドのスクリプト言語。
サーバーサイド ・・・直接ユーザーの目に触れることはなく、サーバー上で処理を行うもの。
.NET（C#, VB） ・・・Windows系アプリケーションを作成できるプログラミング言語。
Java ・・・オブジェクト指向のプログラミング言語。サーバーサイドのコンパイラ言語。プラットフォームに依存しないので汎用性が高い。
Java SE ・・・Javaの基本機能をまとめたもの。パソコン上で動作するゲームや便利なソフトの開発が可能。
Java EE（Jakarta EE） ・・・Java SEを元にしてサーバーサイドの機能を追加したもの。WEBサイトやWEBアプリケーションの開発が可能。
Perl ・・・動的なWEBページを生成することができるプログラミング言語。スクリプト言語の一種。最近はPHPの方が主流。
PHP ・・・動的なWEBページを生成することができるプログラミング言語。サーバーサイドのスクリプト言語。
Ruby ・・・日本で開発されたオプジェクト指向のプログラミング言語。サーバーサイドのスクリプト言語。

■ サーバーソフト
Apache ・・・全世界で最も普及しているWEBサーバーのソフト。オープンソースなので誰でも利用可能。
nginx ・・・Apacheに次いで普及しているWEBサーバーのソフト。Apacheと比べて速い。
IIS ・・・Microsoftが開発したWEBサーバーのソフト。

■ SQL
SQL ・・・データベース言語の一つで、データベースの定義や操作を行うことができる。
DB2 ・・・IBM社が開発・販売しているリレーショナルデータベース管理システム(RDBMS)製品。
MySQL ・・・Oracle社が開発したオープンソースのRDBMS。WEBでよく使われる。
Maria DB ・・・MySQLから派生したオープンソースのRDBMS。
Oracle DB ・・・Oracle社が開発・販売したRDBMS。企業でよく使われている。
PostgreSQL ・・・オープンソースのRDBMS。WEBでよく使われる。最近はMySQLの方が主流。
SQLite ・・・簡易的なシステムでよく使われるオープンソースのRDBMS。
SQL Server ・・・Microsoft社が開発したRDBMS。Windowsのシステムでよく使われる。企業でよく使われている。
CRUD ・・・システムに必要な4つの主要機能である「Create(生成)」「Read(読み取り)」「Update(更新)」「Delete(削除)」の頭文字を並べた用語。
トランザクション ・・・コンピュータシステムにおける、永続的なデータに対する不可分な一連の処理。
コミット ・・・トランザクション処理で結果を確定すること。
ロールバック ・・・トランザクション処理において結果を取り消すこと。
排他制御（排他ロック） ・・・同時アクセスにより不整合が発生することを防ぐために制御すること。
楽観ロック ・・・滅多なことでは他者との同時更新は起きないであろう、という楽観的な前提の排他制御。
悲観ロック ・・・他者が同じデータに頻繁に変更を加えるであろう、という悲観的な前提の排他制御。
実行計画 ・・・どのようにテーブルに対して検索を行うかが書かれた、SQLの実行手順書のようなもの。
チューニング ・・・情報システムやコンピュータ、ソフトウェアなどの設定や構成を調節し、目標の状態に近づけたり、性能を最大限引き出したりする作業
パフォーマンス ・・・コンピュータなどの機器やソフトウェア、システムなどの処理性能や実行速度、通信回線・ネットワークなどの伝送速度・容量などのこと。
スロークエリ ・・・データベースにおいて、時間のかかっているSQLのこと
クエリログ ・・・クライアントからのMySQL Serverへの接続・接続解除の情報、およびクライアントから実行された全てのSQLクエリを出力してくれるログのこと。
IDE（統合開発環境） ・・・プログラムを開発する上で必要なものが詰め込まれた開発環境のこと。
Eclipse ・・・Javaをはじめとする様々なプログラミング言語によるソフトウェア開発に対応した、オープンソースのIDEの一つ。
NetBeans ・・・オープンソースのIDE。Eclipseに比べて扱いやすい。
SpringToolSuite ・・・EclipseをベースにしてSpringFrameworkでの開発に必要なものを組み込んだパッケージ
DBツール ・・・データベース管理タスクに使用できるツール、ユーティリティおよび補助ツールの総称。
A5:SQL Mk-2 ・・・データベース開発において生産性を向上させるべく開発されたフリーのSQL開発ツール。
pgAdmin ・・・GUIによるPostgreSQLの管理ツール。
DBeaver ・・・Eclipseをベースとしたオープンソースソフトで、多数のデータベースに対応し様々な操作を行える。
MySQL Workbench ・・・データベースの設計・開発・管理を一本で実現しているMySQL公式の無料ツール。GUIに対応している。
Sequel Pro ・・・Mac用のMySQLのデータをGUI操作で管理するためのアプリケーション。
TeamSQL ・・・GUIでDBを管理するソフト。現在はサービス終了している。

■ 肩書
PM ・・・プロジェクトマネージャー。プロジェクトの運営、品質、納期等に責任を持ち、プロジェクトを円滑に推進させる役割を果たす。
PL ・・・プロジェクトリーダー。プロジェクトにおけるサブチームのリーダーとして、数名のメンバーの工数、進捗、品質などを管理する。
PG（PE） ・・・プログラマ。SEが作成した設計書に沿ってプログラミングを行う。
SE ・・・システムエンジニア。顧客の要求から使用を決定し、大まかの設計をするまでの情報システム開発における上流工程を担当する。
デザイナー ・・・クライアントや依頼者の求めに応じたデザインを考案する。
エンジニア ・・・工学(エンジニアリング)に関する専門的な知識やスキルを持った人材。
フロントエンド・エンジニア ・・・フロントエンドの設計や構築を行う。
サーバーサイド・エンジニア ・・・サーバーサイドの開発・状態の保存や維持を行う。基幹系企業に使われる事が多い。
バックエンド・エンジニア ・・・サーバーサイド・エンジニアと呼び方が違うだけで内容は大差ない。WEB系企業に使われる事が多い。

■ UI（ユーザーインターフェース）
UI ・・・ユーザーの目に触れる部分や使用される部分。
CUI ・・・Character-based User Interfaceの略。キーボードで操作するUIのこと。
GUI ・・・Graphical User Interfaceの略。マウスや指などで視覚的に操作できるUIのこと。

■ ファイル／ディレクトリ
ファイル ・・・パソコンでデータを扱うときの基本単位。データのまとまりのこと。
拡張子 ・・・ファイルの種類を識別するための文字列のこと。
ディレクトリ ・・・フォルダ。ファイルを整理したり、分類したりができる保管場所。
ルート・ディレクトリ ・・・階層型ファイル構造における最上階層のディレクトリのこと。
カレント・ディレクトリ ・・・ユーザーが現時点で作業を行っているディレクトリのこと。
パス ・・・コンピュータ内で特定の資源の所在を表す文字列のこと。
絶対パス ・・・ルートディレクトリから目的のディレクトリまでの道筋を省略せずに全て記述する。
相対パス ・・・カレントディレクトリから目的のディレクトリの相対位置を記述する。

■ ネットワーク
LAN ・・・Local Area Networkの略。コンピュータや通信機器などをケーブルなどで接続し、相互にデータ通信できるようにしたもの。
WAN ・・・Wide Area Networkの略。地理的に離れた地点間を結ぶ通信ネットワーク。通信事業者が設置・運用する回線網のことを指す事が多い。
クライアント ・・・他のコンピュータやソフトウェアから機能や情報の提供を受けるコンピュータやソフトウェアのこと。
サーバー ・・・他のコンピュータ(クライアント)から要求や指示を受け、情報や処理結果を返す役割を持つコンピュータやソフトウェアのこと。

■ 各種プロトコル
HTTP ・・・Hyper Text Transfer Protcolの略。WEBサーバとWEBクライアントの間でデータの送受信を行うために用いられるプロトコル。
FTP ・・・File Transfer Protcolの略。サーバとクライアントの間でファイルの送受信を行うために用いられるプロトコル。
HTTPS ・・・Hyper Text Transfer Protcol Secureの略。HTTPを暗号化したもの。
SSL ・・・Secure Sockets Layerの略。インターネット上におけるウェブブラウザとウェブサーバ間でのデータの通信を暗号化し送受信させる仕組みのこと。
SSH ・・・Secure Shellの略。リモートコンピュータと通信するためのプロトコル。通信は全て暗号化されるので安全。
ポート、ポート番号 ・・・同じコンピュータ内で動作する複数のソフトウェアのどれが通信するかを指定するための番号。
HTTP、FTP、SSHの各ポート番号 ・・・HTTPポート番号：80 FTPポート番号：21 SSHポート番号：22
ファイアウォール ・・・ネットワークの境界に設置され、内外の通信を中継・監視し、外部の攻撃から内部を保護するためのソフトウェアや機器、システムなどのこと。
TCP ・・・Transmission Control Protcolの略。送ったデータが相手に届いたかその都度確認しながら通信する。信頼性は高いが転送速度が低い。

■ IPアドレス
IPアドレス ・・・インターネット上に接続された機器が持つナンバー。
グローバルIPアドレス ・・・インターネット上で通信を行う際に必要。重複不可。
プライベートIPアドレス ・・・組織内のネットワークでのみ使用できるIPアドレス。組織内での重複は不可。

■ ドメイン
ドメイン ・・・インターネット上の住所にあたる。
DNS ・・・Domein Name Systemの略。ドメインとIPアドレスを対応付けて管理するシステムのこと。
VPN ・・・Virtual Private Networkの略。インターネット上に仮想の専用線を設け、安全なルートを確保した上で重要な情報をやり取りする事ができる。
ブリッジ ・・・ネットワーク中のデータを見て、行き先を振り分けてくれる機器のこと。
プロキシサーバー ・・・インターネットに直接接続できないコンピューターに代わり、インターネットに接続しWebサイトへのアクセスなどを行うサーバーのこと。

■ 電子メール
SMTP ・・・Simple Mail Transfer Protcolの略。メールを相手のメールサーバーまで届ける仕組み。
POP ・・・Post Office Protocolの略。メールサーバーから受信したメールを受け取る仕組み。
IMAP ・・・Internet Message Access Protocolの略。POPと同じくメールを受け取る仕組み。POPと違いメールをサーバーに残せる。

■ セッションとクッキー
セッション ・・・接続を確立してから切断するまでの一連の通信のこと。
クッキー ・・・ホームページを訪問したユーザーの情報を一時的に保存する仕組み。
キャッシュ ・・・訪問したホームページの情報を一時的に保存する仕組み。

フロー ・・・コンピュータプログラムを構成する命令や手続き、処理の流れなどのこと。

■ バージョン管理
Git ・・・オープンソースの分散バージョン管理システムの一つ。
SVN ・・・Apache Subversionのこと。ファイルの変更履歴を管理するシステムの一つ。
デグレ ・・・デグレード。以前よりプログラムの状態が悪くなること。
マージツール ・・・結合の作業を効率化するツール。
